Writing lean instructions
✓ Best practices
  • One purpose per folder — never mix inputs with outputs
  • Always save outputs to the correct subfolder, never to the root
  • Use lowercase and hyphens, no spaces in file names
  • Never overwrite input files
  • Keep Global rules universal — project-specific rules go in Layer 2 or 3
  • Skip long explanations of why in .md files — just the rule
  • If a rule only applies to one task, put it in the message, not CLAUDE.md
⚡ Token-efficient rules
❌ Too verbose"When the user gives you a document to transform, please make sure you always check what format they want before starting..."
✅ Efficient"If format is not specified, ask: 'Which format? thread / post / summary / pdf / slides'"
Every line in your .md files is loaded into Claude's context. Write like you're paying per word — because you are.
